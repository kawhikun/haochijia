# 好吃家 v26 补丁说明（2026-04-18）

## 本轮对应用户反馈的修复

### 1. 音乐区按钮仍然像没反应
- 把音乐区交互补成 `click + touchend` 双通道事件委托，专门兜移动端点按。
- 强化了流派按钮与风格模板的选中态、边框、焦点态和层级，减少“其实点到了但看不出来”的情况。
- 音乐区说明文案缩短：保留模板和 8 参数细调，但把冗长开发者式说明移出主界面。

### 2. 3D 模型画面不够突出
- 提高 3D 舞台高度，并加重灯光、镜头贴近程度、环光、阴影和背景氛围。
- WebGL 场景与 2D fallback 都同步增亮、放大主体，减少模型显得小和扁的情况。
- 首页继续把营养进度放在舞台下方，不再压住模型主体。

### 3. 浅色字 + 浅色框看不清
- 统一加深次级文字颜色、边框强度、输入框对比度。
- 提升 ghost button、标签、状态胶囊、音乐卡片、搜索卡片等浅底组件的可读性。

### 4. 搜索结果里有些字像竖着排
- 搜索结果的关键文本改为正常横向换行，不再用过于激进的 `overflow-wrap:anywhere` 行为。
- 份量胶囊改成单行展示；移动端结果头部改成更稳的单列布局。

### 5. 首页要更突出 3D 人体 + 搜索食品并输入营养 + 营养进度
- 首页新增“手动营养输入”主按钮。
- 首页新增独立的实时营养进度板，和当天摄入 / 未保存手动预览联动。
- 剩余按钮移进“探索更多功能”折叠区，减少主界面拥挤。

### 6. 音乐区文字过多，诉求 / 规划要移到文件里并自动整理
- 新增 `user_request_plan_log_v26.json` 作为源数据。
- 新增 `sync_request_plan_log.py`，会把 JSON 自动整理成 `Haochijia_User_Requests_And_Plan_v26.md`。
- `run_local_server.py` 已接入自动同步：每次本地启动前会先重建这份 Markdown。

### 7. 缓存问题
- Service Worker 版本升级到 v26，减少旧缓存继续顶住新版资源的概率。

## 本轮验证
- `assets/app.js` 已通过 `node --check`
- `assets/body-module.js` 已通过 `node --check`
- `run_local_server.py` 与 `sync_request_plan_log.py` 已通过 `python -m py_compile`
