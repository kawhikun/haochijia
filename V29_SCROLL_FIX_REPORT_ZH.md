# 好吃家 v29 滚动稳定性与体验修复说明

## 这次重点修了什么

### 1. 移动端抽屉滚动被抢的问题
- 收紧了 `assets/app.js` 里 v28 的字段保可见逻辑：
  - 不再把所有 `input` 都当成要强行保可见的对象。
  - 现在只追踪真正的文本录入字段（如 `text / search / number / textarea / select`）。
  - `range / checkbox / radio` 不会再触发抽屉滚动修正。
- 结果：音乐页的滑杆不再更容易把滚动手势抢掉；营养录入里的数字输入也不会被过度干预。

### 2. 音乐页“拉不了”的触摸误判
- 重写了音乐面板的流派 / 模板点击委托：
  - 增加了触摸位移阈值。
  - 只有真点击才触发切风格；拖动 / 滑动不再误判成点选。
- 结果：音乐区向上 / 向下拖动更稳，不容易因为手指落在流派卡片上而被吞掉。

### 3. 抽屉和快捷面板的滚动容器更稳
- 强化了 `assets/styles.css`：
  - `v22-sheet-body` 强制 `overflow-y: auto`、`overflow-x: hidden`。
  - 增加 `overscroll-behavior-y: contain`、`-webkit-overflow-scrolling: touch`。
  - `quick-sheet-body` 也补了明确的可滚动高度与触摸滚动约束。
- 结果：底部抽屉、快捷面板都更接近“只在内部滚，不把整页带飞”。

## 额外推进的体验改进

### 4. 营养手动录入更清楚
- 新增“本次准备加入什么”实时预览卡：
  - 显示本次录入会加进去的热量 / 蛋白质 / 碳水 / 脂肪等。
  - 区分 OCR 回填与手动录入预览。
  - 强调“预览不等于已保存”，避免误会。

### 5. 食品搜索更有权威感
- 新增食品库依据条：
  - 当前激活库（中国优先 / 国际包装食品优先）
  - 可搜索条目数
  - 数据日期
  - 翻译层信息
- 结果：用户更容易知道当前搜索到底依赖哪套数据、适合搜什么。

### 6. 音乐面板更好玩
- 新增一排一键情绪捷径：
  - 慢逛 / 专注 / 推进
- 用现有的风格预设直接切换，减少层层点选。

### 7. 视觉层做了轻度抛光
- 音乐状态条改成更像“实验室状态卡”。
- 食品库依据条、手动预览卡都改成更像信息模块，而不是孤零零的一行小字。
