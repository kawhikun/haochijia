# 好吃家 v30 核心强化版（2026-04-21）

## 本轮目标

1. 全面检查 v22 / v23 / v25 / v26 / v27 / v28 / v29 clean 线中的优点并收口到一个新版。
2. 主文件只保留两条主线：
   - 身体数据记录：用户状态与维度输入、营养建议、3D 模型
   - 营养摄入分析：食品库、营养进度、拍照分析
3. 音乐功能从主文件拆出，形成独立 `music-lab.html`。
4. 首屏直接呈现 3D 人体模型，并在人体周围保留 6 条营养环。
5. 解决“输入数据后界面突然跳走”的问题。
6. 增加一键导入身体维度、一键导出全部记录、本地加固保存和 GitHub 备份入口。

## 核心诊断

### 1) 用户输入后界面突然跳走
已经定位到是两层自动收起叠加导致：

- `assets/app.js` 在收到 `haochijia:body-sync` 且 `saved=true` 时调用 `closeV22Sheet()`，身体保存后直接把抽屉关掉。
- `assets/body-module.js` 在 `saveBodyRecordV20()` 里又执行了 `setBodyPanelState(false)`，再次把身体面板收起。

### 2) 3D 营养环虽已存在，但还不够符合“核心版”目标
- 旧版营养环重点项还不够自适应。
- 旧版移动端首页仍保留音乐 / 套餐 / 数据等旁支入口。

## 本轮已做改动

### A. 主页收口为核心版
- 新增 `assets/core-upgrade-v30.js`
- 新增 `assets/core-upgrade-v30.css`
- 主站隐藏：启动引导、折叠模块中枢、概览大看板、音乐页、组合餐页、今日页、数据说明页。
- 首页新增“核心控制台”，只保留：
  - 身体数据
  - 营养摄入
  - 拍照分析
  - 导入维度
  - 导出全部
  - GitHub 备份
  - 音乐实验室入口（跳转独立页）

### B. 6 条营养环升级为“基础三环 + 自适应三环”
- 固定三环：蛋白质 / 脂肪 / 碳水
- 自适应三环按用户状态动态切换：
  - 骨关节：钙 / 维生素 D / 维生素 K
  - 贫血：铁 / 叶酸 / 维生素 B12
  - 血压：钠 / 钾 / 饮水
  - 血脂：饱和脂肪 / 膳食纤维 / Omega-3
  - 血糖：膳食纤维 / 蛋白质 / 碳水
- 首页新增“当前模式 / 当前 6 环”说明条，方便用户理解为什么这 3 个环会突出。

### C. 修掉保存后跳走
- `app.js`：身体保存成功后不再执行 `closeV22Sheet()`。
- `body-module.js`：身体保存后不再把面板折叠关闭，只保留 action hub 收起。
- 核心版同时隐藏“保存并收起”按钮，避免用户再次被误导。

### D. 3D 触屏交互增强
- 舞台区域启用更明确的触控提示。
- 双击人体：回到正视图。
- 长按人体：直接打开身体数据。
- 放大 3D 视角按钮、舞台触发按钮和人体快捷按钮的触摸面积。
- 营养环优先项增加高亮和标签强调。

### E. 导入 / 导出 / 本地加固保存 / GitHub 备份
- 一键导入身体维度：支持 JSON / CSV
- 一键导出全部：导出全部 `haochijia.*` 本地记录快照
- IndexedDB 本地镜像：把站内关键数据镜像一份，便于同源更新后自动恢复
- 浏览器持久化存储申请：尽量降低本地数据被系统清掉的概率
- GitHub 备份面板：支持填写 owner / repo / branch / path / token 后上传或恢复

### F. 音乐拆出独立页
- 新增 `music-lab.html`
- 新增 `assets/music-lab.js`
- 新增 `assets/music-lab.css`
- 主站里的音乐按钮与人体快捷入口改为跳转音乐实验室
- 音乐实验室保留：
  - 流派选择
  - 8 参数滑杆
  - 持续播放 / 自动变形开关
  - 桥接包导出
