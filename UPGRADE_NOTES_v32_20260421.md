# 好吃家 v32 核心全面强化版（2026-04-21）

## 这版怎么做的
- 基于“只保留核心功能”的版本继续强化，不把音乐、装饰页、无关入口加回来。
- 首页继续是 3D 人体舞台，围绕模型展示 6 条主营养环。
- 延续 v31 已修复的“保存身体记录后突然跳走”处理，保存后留在当前流程。

## 这次实际加强了什么
### 1) 审美与首页舞台
- 重新按配色方案做了深色渐层舞台、玻璃卡片、品牌高光、平台感区分。
- iPhone / Android 两类设备的首页舞台做了细微区分：光影、镜头高度、运动感不同。
- 6 条营养环重新布局到模型外围，尽量避免遮挡人体主体。

### 2) 3D 模型与互动
- 保留拖动旋转、双指缩放、双击回正、长按直达身体记录。
- 调整了灯光、舞台光环、雾效、镜头和自动旋转速度，让人体更立体。
- 营养环高亮会继续驱动模型聚焦区域与舞台主色。

### 3) 照片塑形
- 继续保留上传全身照参考 + 六维塑形：肩、胸、腰、臀、上肢、腿。
- 自动估测后可继续手动微调。

### 4) 食品库双库和翻译
- 改成真实双库加载，不再是“看起来分中文/国际，实际底层还是一整包”。
- 中文库：36,793 条
- 国际库：73,117 条
- 支持中文 / EN / 原始名三种显示模式，并在搜索里同时匹配中文、英文、原始名、品牌、条码。

### 5) 数据层
- 本地继续优先：localStorage + IndexedDB + 浏览器持久化申请。
- 支持完整快照导出、身体记录 JSON、身体记录 CSV、饮食日志 CSV。
- 保留最近本地备份恢复。

### 6) GitHub 同步
- 默认预填：kawhikun / haochijia / data/haochijia-user.json
- 继续保留手动同步、恢复和自动同步开关。
- 设计策略仍然是“本地优先，GitHub 云端作为补充同步”。
