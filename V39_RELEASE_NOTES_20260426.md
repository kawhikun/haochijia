# Haochijia v39 - Standard Body Model Update

日期：2026-04-26

本版本只优先处理用户要求的人体模型与相关联动，不再扩展食品库等非本次重点模块。

## 本次完成

1. 接入上传的标准人体模型资源
   - 从用户上传的“男女人体标准模型”资源中提取标准男/女体型网格。
   - 新增 `assets/body-models/standard-bodies.json`，前端 WebGL 会优先加载该标准人体模型。
   - 若标准模型加载失败，保留原有 Canvas/WebGL 程序化模型作为兜底，不影响记录功能。

2. 身体维度实时体现在模型中
   - `sex` 会传入模型，自动切换标准男/女体型。
   - 身高、体重、体脂、肩宽、胸围、腰围、腹围、臀围、上臂、前臂、大腿、小腿、脚踝会参与模型塑形。
   - 照片捏形参数继续叠加到肩、胸、腰、臀、手臂、腿部。
   - 模型界面不会显示用户年龄；年龄只留在基础数据中供营养目标计算使用。

3. 基础数据和身体维度可折叠
   - 用户基础数据卡片增加“收起 / 展开”。
   - 身体维度卡片增加“收起 / 展开”。
   - 保存身体维度后，如果基础数据和核心维度已填写，会自动收起对应卡片，突出模型展示。

4. 身体维度按自然日更新
   - 身体记录新增 `recordDate`、`dayStart`、`dayEnd`、`updatedAt`。
   - 同一天 0:00-23:59 内再次保存身体维度，会更新当天记录，而不是无限追加重复记录。
   - 身体历史按自然日展示，并支持全量导出。

5. 摄入记录按自然日保存
   - 食品、饮水、拍照识别加入摄入记录时，都会写入 `recordDate`、`dayStart`、`dayEnd`。
   - 每天记录以本地自然日 0:00-23:59 为边界。
   - CSV 导出增加自然日字段，保留所有历史摄入条目。

6. 营养建议增加历史摄入参考
   - 建议卡片在今日摄入基础上，加入近 7 天有记录日的平均摄入和达标天数。
   - 建议依据不再在可视建议区展示“年龄分组”。
