# 好吃家版本优点总检（v22 → v30）

## v22 / v23 线
### 主要优点
- 把“3D 人体 + 身体记录 + 食品记录 + 拍照识别 + 音乐”首次整合进同一套移动端界面。
- 双食品库检索路线清晰：中国库 + 国际库联动。
- 条码 / OCR / 今日记录抽屉都已经成型。
- 3D 人体舞台已经具备明显主视觉潜力。

### 本轮继承
- 双库搜索与来源区分
- 身体模块与主营养模块的联动骨架
- 拍照识别链路

## v25
### 主要优点
- 首次搜索卡顿明显改善。
- 音乐风格模板更丰富。
- 营养进度不再粗暴压住 3D 人体主体。

### 本轮继承
- 首页必须先保证“能搜、能记、不卡”。
- 舞台周围只保留高价值信息，不再堆满遮挡层。

## v26
### 主要优点
- 文字可读性更稳。
- 3D 模型视觉权重更强。
- 首页主线开始向“3D + 搜索 + 营养进度”收拢。

### 本轮继承
- 主文件必须减法设计，先保护核心任务完成率。

## v27
### 主要优点
- 底部抽屉和移动端视口稳定性大幅增强。
- 营养环围绕人体的方向是对的。
- 实时营养联动已经具备基础形态。

### 本轮继承
- 6 环继续保留，但从通用 4 环升级为“基础三环 + 自适应三环”。
- 抽屉和键盘策略继续沿用更稳的移动端方案。

## v28 / v29 clean
### 主要优点
- Pages-ready 结构更干净，适合直接部署。
- 视口、键盘、触控、样式都比早期版本更成熟。
- clean 包更适合作为新版本基底。

### 本轮继承
- 以 clean 线为基底构建 v30。
- 保留 Pages 友好的目录结构与缓存策略。

## v30 取舍原则
### 保留的
- 身体数据记录
- 用户状态与营养建议
- 3D 模型
- 食品库
- 营养进度
- 拍照分析
- 本地保存

### 分离的
- 音乐实验室

### 新增强化的
- 6 条自适应营养环
- 保存不跳走
- 3D 双击 / 长按增强
- 一键导入身体维度
- 一键导出全部记录
- IndexedDB 镜像恢复
- GitHub 云端备份
