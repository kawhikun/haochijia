# 好吃家 v26 用户诉求与实施规划同步记录

更新日期：2026-04-18

## 用户诉求总表
1. 首屏流程要更清楚，移动端优先。
2. 音乐、身体、3D 人体、食品搜索与营养进度要串成单流程。
3. 音乐必须可点可调，不同 Techno / 电子元素要能细分强化。
4. 首次搜索食品库不能卡住主线程。
5. 3D 人体模型要更突出，不能被别的面板抢视觉。
6. 每日营养进度要和食品搜索、手动输入、OCR 结果实时联动。
7. 移动端顶部和其他次级功能不要长时间占据主界面。
8. 浅色背景 + 浅色文字的组合要统一修掉。
9. 搜索结果中文字不能出现难辨识的竖排 / 极端换行。
10. 首页应突出 3D 人体、搜索食品并输入营养、每日营养进度；剩余功能折叠为探索入口。
11. 音乐界面的开发者说明太多，要移出 UI，放到单独文件。
12. 用户诉求与实施规划要自动整理成文件。

## 当前实施规划
1. 音乐区增加点击兜底、压缩文案、强化选中态与风格模板。
2. 首页主任务区只保留搜索、手动营养输入、拍照 / 条码和实时营养进度。
3. 身体主舞台提高高度、灯光、镜头和阴影强度，让模型更大更亮。
4. 所有轻底色组件统一提高文字对比度和边框强度。
5. 搜索结果为关键文本启用正常横向换行，并把份量胶囊改为单行。
6. 把诉求 / 规划记录移到 JSON + Markdown，同步脚本在本地启动时自动重建。
