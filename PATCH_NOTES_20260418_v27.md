# 好吃家 v27 补丁说明（2026-04-18）

## 本轮目标
本轮针对 5 个高优先级问题继续收口：
1. 修掉手机端底部面板偶发掉到屏幕下方、只剩一条关闭头部可见的问题。
2. 让 3D 人体模型重新成为主视觉，不再被锁定文案和大段说明遮住。
3. 把每日营养进度改成围绕人体的多层营养环可视化。
4. 再做一轮全局稳定性、文字排版和可读性加固。
5. 自动整理最新的用户诉求、实施规划和主提示词文件。

## 已完成改动

### 1) 移动端底部面板稳定性
- `openV22Sheet()` 继续收紧打开流程：先重置页面滚动、抽屉滚动，再做容器内滚动。
- 抽屉内部不再依赖高风险 `smooth scrollIntoView()`，改为更稳的容器内滚动。
- 移动端自动聚焦统一取消：`safeFocusV27()` 在移动端直接跳过，减少键盘弹出把固定抽屉挤坏的概率。
- `body-module.js` 打开身体面板时，优先滚动最近的 `.v22-sheet-body` 容器，而不是让页面整体跳动。
- 补充了底部抽屉的安全区、`svh/dvh` 高度、`overscroll-behavior` 和 `flex-shrink` 约束。

### 2) 3D 人体模型舞台
- 3D 舞台新增 `v27NutritionOrbitStage`，在人体周围显示多层营养环。
- 营养环默认覆盖：热量、蛋白质、膳食纤维、饮水。
- 营养环支持点击，直接引导到对应的营养输入 / 食品搜索动作。
- 移动端人体舞台不再叠加锁定芯片、浮层说明、热力图图例等次级覆盖层。
- 缺失围度从“锁定文案”改成静默短横线，减少“模型被文字锁住”的感觉。

### 3) 实时营养联动
- 营养环直接读取实时营养快照，兼容食品搜索添加、手动营养输入、OCR / 拍照识别预览等来源。
- 当存在未保存预览时，舞台底部会显示轻提示，说明营养环已按实时预览更新。

### 4) 可读性与反竖排修复
- 再次统一提高浅色卡片、按钮、标签、次级文案的文字对比度。
- 搜索结果中的标题、来源、标签、份量胶囊、元信息统一强制横向排版。
- 移动端搜索结果元信息改成更稳的单列 / 双列栅格，减少窄宽度挤压造成的竖排错觉。
- 份量胶囊保持单行，避免被压成难以辨识的窄条。

### 5) 自动整理用户诉求与 Prompt
- `sync_request_plan_log.py` 已升级为自动读取最新 `user_request_plan_log_v*.json`。
- 现在会自动生成两份文件：
  - `Haochijia_User_Requests_And_Plan_v27.md`
  - `Haochijia_Master_Prompt_v27.md`
- 新增 `user_request_plan_log_v27.json`，把本轮核心诉求、计划、已落实项、限制与主提示词一并纳入版本化记录。

### 6) 缓存与版本
- Service Worker 缓存版本升级为 `haochijia-v27-orbit-stability-20260418-r1`。
- 注册 URL 也升级到 `sw.js?v=20260418-v27`，降低旧缓存覆盖新版资源的概率。

## 仍需诚实说明
- 当前环境无法直接做真实手机浏览器的触摸 / 键盘 / 视口缩放回归，因此本轮稳定性提升主要来自代码级兜底与滚动 / 聚焦策略收紧。
- 如果设备浏览器的 WebGL、远程 ESM 或网络环境受限，3D 远程模型仍可能回退到本地 fallback 方案。
