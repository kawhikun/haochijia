# 好吃家 v25 用户直改补丁（2026-04-18）

## 这次针对你提的 5 个问题，直接改了什么

### 1) 首次搜索食品库很卡
- 把 `Promise.all([当前库, 兜底库])` 改成了 **当前库先返回结果，兜底库再后台补齐**。
- 食品库预处理改成了 **分批让出主线程**，减少第一次整理大库时的长时间卡顿。
- 搜索开始时会立即给出“正在读取哪个库”的反馈，不再像死掉一样没反应。

### 2) 音乐只能点 techno、其他按钮没反应 / 风格不够细
- 首页食品入口卡片新增了 **“音乐实验室”** 快捷入口。
- 音乐区新增 **风格强化器 / Style Macros**：
  - 催眠循环
  - 仓库重压
  - Acid 线条
  - Dub 雾面
  - 旋律推进
  - 碎拍弹性
- 这些风格模板会直接改动流派 + 8 个音乐参数，但下面的 8 个滑杆仍然保留，你可以继续细调。
- 风格卡和流派卡都补了更适合移动端的点击反馈。

### 3) 3D 模型还是被每日营养进度挡住
- 把原来压在 3D 舞台上的 **营养进度浮层** 改成了 **舞台下方的营养卡片区**。
- 身体围度分区芯片仍保留在模型周围，但移动端缩小并重新排位，不再那么挡。
- WebGL 3D 模块不再只依赖一个远程源，会按顺序尝试多路模块地址，提高漂亮 3D mesh 拉起成功率。
- 如果 WebGL 还是失败，会回到本地轮廓模式，但现在不会再被营养进度遮住。

### 4) 每日营养进度要和用户输入实时联动
- 修掉了 v22 舞台营养进度读取错对象的问题：之前卡片用的是 `snapshot[item.id]`，实际应该读 `snapshot.totals[item.id]`。
- 手动营养录入新增 **未保存实时预览**：
  - 改营养字段
  - 改克重 / 份数
  - OCR 解析后回填
  都会立刻反映到“今日营养进度”和舞台营养卡片里。
- 保存后会自动退出预览态，避免重复叠加。

### 5) 移动端顶部大框一直占着顶部
- 顶部 `好吃家营养台` 头部增加了 **滚动折叠 / 隐藏** 行为：
  - 轻微下滑：缩成紧凑头部
  - 继续下滑：自动收起
  - 往上滑：重新出现

## 我额外顺手补的点
- 营养卡、音乐卡、首页快捷按钮都补了更适合触屏的 `touch-action`。
- 首页入口按钮从 4 个扩成 5 个：身体 / 音乐 / 套餐 / 上传 / 数据。

## 还需要你知道的一点
- “漂亮 3D mesh” 这块我已经把 **遮挡问题** 和 **单 CDN 脆弱性** 一起处理了。
- 但如果用户设备当时完全离线、或浏览器本身禁掉了远程 ESM / WebGL，那仍会进入本地轮廓模式。这不影响记录与联动，只影响 mesh 的视觉级别。
