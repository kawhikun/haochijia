# 好吃家 v22 部署说明

## 适用场景

适用于：
- GitHub Pages
- 静态网站托管
- 直接把站点文件上传到网站根目录

## 推荐部署包

使用 `haochijia-v22-root-ready-web-upload.zip`。

解压后，把 `haochijia-v22-root-ready-web-upload` 文件夹里的全部内容上传到站点根目录。

## 为什么这次必须覆盖旧文件

本项目使用了 Service Worker 和本地缓存；如果只是部分替换文件，旧 HTML / 旧 JS / 新资源可能会混在一起，继续造成：
- 按钮像没反应
- 页面还是旧版布局
- 搜索和抽屉逻辑异常
- 音乐逻辑表现不一致

## 推荐部署步骤

1. 删除线上旧文件，或确认会被完整覆盖
2. 上传 v22 根目录文件
3. 首次打开时强制刷新一次
4. 若浏览器仍明显停留在旧版本，关闭标签页后重新打开站点

## 版本识别点

部署成功后，应该看到：
- 首屏先弹出 Apple / Android / Desktop 分流
- 之后自动进入音乐设置抽屉
- 首页只剩 3D 人体主舞台和食品入口卡片
- 搜索不到时会自动跨中国库 / 国际库补搜

## 本版不应再出现的典型旧症状

- 一整页堆满很多功能卡片
- 食物输入必须先去找隐藏区块
- 身体和营养参数分散在不同地方
- 收起来后找不到重新进入路径
