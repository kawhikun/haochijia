# 好吃家 v31 核心重构说明

## 保留并强化的核心
- 身体数据记录：用户状态、维度输入、营养建议、3D 人体模型
- 营养摄入分析：食品库、营养进度、拍照分析
- 首页直达 3D 人体舞台，6 条营养环围绕模型实时更新
- 状态驱动营养环：骨关节 / 跟骨 / 膝 / 骨密度等输入会前置钙、维生素 D、维生素 K

## 从旧版本中保留的优点
- v23：本地保存更稳，适合做离线兜底
- v28：Pages 发布结构更干净，适合直接部署
- 20260417~20260419：逐步补齐身体模块、食品库、营养进度和拍照识别
- 20260419：3D 人体与营养算法最完整，因此被作为这次重构的核心素材来源

## 这次重构实际删掉了什么
- 音乐相关页面和入口
- 非核心说明文字
- 历史叠加的首页跳转、工作区切换和多余按钮
- 与主流程无关的装饰页和旁路入口

## 已修问题
- 保存身体记录后不再强制跳回首页
- 3D 模型支持拖动旋转、双指缩放、双击回正、长按直达身体记录
- 支持导入身体维度 CSV / JSON
- 支持导出全部记录快照
- 增加 IndexedDB 镜像和持久化存储申请
- 增加 GitHub 云端备份 / 恢复面板

## 关于照片塑形
- 当前实现是“上传全身照片作为参考 + 四个维度滑杆塑形”
- 这是适合静态 Pages 环境的稳定实现
- 不是医学级单张照片自动 3D 重建，但已经能让用户按自己照片快速捏出更接近的体态
