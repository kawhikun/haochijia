# v41 修正说明 - 2026-04-26

本版只处理人体模型卡死与 3D 模型轻量化问题。

## 修复原因

v40 在模型控制器初始化时先创建了 `canvas.getContext('2d')`，随后才尝试把同一个 canvas 交给 Three.js 创建 WebGL 渲染器。部分浏览器会因此无法再创建 WebGL 上下文，导致页面进入 2D fallback。v40 的 2D fallback 又会对完整标准人体网格做三角面排序和绘制，因此打开页面或输入身体数据时可能卡死。

## 本版改动

1. 模型控制器改为优先创建 WebGL，不再提前占用 2D canvas context。
2. 2D fallback 改成惰性启用：只有 WebGL 确认不可用时才创建 2D context 和绘制 fallback。
3. 标准男 / 女身体网格保留，但改为轻量标准网格：
   - 女性：4186 顶点 / 8368 三角面 -> 1667 顶点 / 3361 三角面
   - 男性：8830 顶点 / 17656 三角面 -> 2453 顶点 / 4973 三角面
4. WebGL 渲染降频到最高约 30fps，低功耗 / 减少动态模式约 18fps。
5. 关闭模型阴影、关闭抗锯齿、降低 DPR 上限，减少移动端 GPU 压力。
6. 模型输入更新加入轻微 debounce，避免输入每个数字时立即反复销毁 / 重建网格。
7. 移动端默认不叠加历史幽灵模型，避免双模型同时渲染。
8. 保留 v40 已做的要求：模型界面不显示年龄，不显示体重 / BMI / 腰臀比等六个遮挡营养环的浮层；身体维度仍会驱动标准人体模型塑形。
