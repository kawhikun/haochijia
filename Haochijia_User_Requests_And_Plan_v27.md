# 好吃家 v27 用户诉求与实施规划同步记录

更新日期：2026-04-18

## 用户诉求总表
1. 手机打开时，任何底部面板都不能突然掉到屏幕外，更不能只剩一条可见的关闭按钮。
2. 3D 人体模型必须始终是主视觉，输入数据后不允许再被锁定文案、围度缺失提示或大段文字遮住。
3. 每日营养进度要参考运动环的感觉，用几层围绕人体的营养环做实时可视化。
4. 食品搜索、手动营养输入、OCR / 拍照识别写入后，营养进度必须立刻联动。
5. 首页必须突出 3D 人体、搜索食品并输入营养、营养进度，剩余功能自动折叠为探索入口。
6. 音乐区域仍需保留可调性，但文案要短、按钮要稳、用户一触就有反馈。
7. 所有浅色框 + 浅色字、竖排字、极端换行、难辨识文本都必须清理干净。
8. 所有用户操作都要优先稳定，避免点击无响应、滚动乱跳、面板错位、输入后界面失稳。
9. 每次迭代都要自动整理用户诉求、实施规划和主提示词文件。

## 当前实施规划
1. 移动端统一取消自动 focus 和高风险 smooth scroll，优先使用容器内滚动和无跳动打开策略。
2. 底部抽屉统一加安全区、固定高度、overscroll 约束，减少键盘弹出或视口变化导致的错位。
3. 3D 舞台改为营养环主视觉，移除锁定文案，移动端只保留人体和营养环，不再叠大量说明层。
4. 身体围度标签只显示已有数据的项目，没有数据时不再在人体周围摆出锁定提示。
5. 搜索结果、按钮、标签、胶囊和次级文案统一增加对比度并强制横向排版。
6. 本地同步脚本自动从最新 JSON 生成诉求 / 规划 Markdown 和主提示词 Markdown。

## 本轮已落实
1. openV22Sheet 改为移动端更稳的打开流程：先重置页面与抽屉滚动，再使用容器内滚动，不再对抽屉内部节点使用高风险的 smooth scrollIntoView。
2. 移动端自动聚焦被统一收紧：safeFocusV27 在移动端直接跳过，身体模块滚动到表单时也不再自动弹起键盘。
3. 身体模块改为优先滚动最近的 v22-sheet-body 容器，减少内容被滚到屏幕底部或抽屉只剩一条头部的概率。
4. 3D 舞台新增多层营养环 SVG 叠层，围绕人体显示热量、蛋白质、膳食纤维和饮水进度，并支持点击营养环跳到输入 / 搜索。
5. 移动端移除了人体周围的锁定芯片、浮层说明、热力图图例等次级文字覆盖；缺失围度显示改为短横线，不再显示锁定提示。
6. 全局对比度再提升一轮，并对食品卡、标签、音乐芯片、说明文字、搜索结果元信息启用更稳的横向断行规则，减少竖排和极端换行。
7. Service Worker 缓存版本升级到 v27，降低旧缓存继续顶着旧样式和旧脚本的概率。
8. 同步脚本已升级为读取最新 user_request_plan_log_v*.json，并自动生成 v27 的诉求 / 规划文件和主提示词文件。

## 仍需说明 / 限制
1. 当前环境无法直接打开真实手机浏览器做触摸、键盘、视口缩放的完整回归测试，因此这轮稳定性主要依赖代码级兜底、滚动策略收紧和缓存版本升级。
2. 远程 3D 资源与 WebGL 能力仍取决于设备、浏览器和网络条件；当远程 mesh 或 WebGL 不可用时，页面仍会退回本地轮廓 / fallback 方案。
